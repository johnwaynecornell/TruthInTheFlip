Note on RandomSD same-source controls:

The RandomSD NET2 run should be understood as a same-provider randomized anticipation control, not as a fully independent external predictor. The flip stream and the RandomSD anticipation stream are both divided from the same cryptographic randomness service, but the runner still preserves fair temporal order: the prior anticipation is scored against the current flip before the next anticipation state is computed. In that sense, RandomSD does not know the current answer before the guess is judged; it is another consumer of the same washed entropy source.

For NET2 this is acceptable and useful because cryptographic whitening is specifically designed to erase usable relation between draws. Any dependency is therefore micro-level — implementation order, buffering, thread scheduling, provider state — rather than a macro-level predictive channel. This makes RandomSD a good “same-source noise control,” while leaving room for a separate distinction in future QRNG tests between external-random controls, same-source controls, and serialized raw-entropy controls.
